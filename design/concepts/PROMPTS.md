# Final concept prompt set

Mode: built-in Image Generation.

The current v183 home/cave/forest/aurora/space frames and the licensed Kenney
contact sheet were used as visual references. Later prompts used the earlier
generated screens as binding style references so the set stayed coordinated.

## 00 — Home and progression

Create a complete 9:16 SKYSTACK home screen in the same premium pixel system as
the cave concept. Preserve the recognizable logo, colorful block tower, mascot,
and simplicity. Show a restrained vertical world preview from cave and forest
through clouds, aurora, and space. Use one centered next-level card and one large
`CONTINUE` action, plus compact `HOME`, `MAP`, and `ME` bottom navigation. Use only
the exact visible copy `SKYSTACK`, `STACK TO THE STARS`, `CONTINUE`,
`THE FOREST`, `99M → 180M`, `HOME`, `MAP`, and `ME`. Avoid dashboard clutter,
popups, competing calls to action, tiny stats, overlaps, and blur.

## 01 — Caves vertical slice

Create a complete 9:16 CAVES gameplay screen using licensed pixel-platformer and
dungeon vocabulary. Build continuous near/mid/far cave strata with mine braces,
crystals, roots, fossils, mushrooms, lantern pools, dust, bats, insects, and a
distant tunnel, while preserving a broad, empty central stacking lane. Keep the
colored tower, mascot, sliding block, cut edges, and horizontal alignment as the
highest-contrast elements. Use crisp 16–18 px vocabulary with only restrained
smooth lighting behind it. Show only `CAVES` and `GOAL 99M`. Avoid a full-screen
veil, central clutter, painterly or 3D rendering, large panels, and overlapping
text.

## 02 — Forest wind vertical slice

Create a complete 9:16 THE FOREST gameplay screen in the same system. The climb
has emerged from a cave into authored soil, roots, trunks, layered canopy,
distant treeline, dappled sun, mushrooms, ferns, flowers, a nest, a small ruin,
butterflies, and two habitat-owned birds. A left-to-right gust must be anticipated
by leaves, grass, vines, and seeds before it affects the block. Keep the center
lane empty and the tower dominant. Show only `THE FOREST` and `GOAL 180M`. Avoid
fog veils, bokeh, wildlife in the landing lane, and decorative motion unrelated
to the real wind force.

## 03 — Aurora and balloon vertical slice

Create a complete 9:16 AURORA gameplay screen showing the vertical transition
from stratosphere to polar night and space. Render translucent pixel-edged aurora
folds at distinct world heights so the player rises through them. Add ice
crystals, a curved Earth horizon, sparse cloud remnants, and a few upper-edge
satellite silhouettes. Put a clearly identifiable repair balloon in a side route
with envelope, ropes, basket, sway, lantern, and wrench emblem; it must not cross
the active lane. Show a short ice-slide afterimage on an off-center block. Use
only `AURORA` and `GOAL 615M`. Avoid hard comb bars, screen-anchored veils,
balloon spam, central power-ups, and excessive bloom.

## 04 — Space asteroid vertical slice

Create a complete 9:16 `SPACE II` / `ASTEROID FIELD` gameplay screen. Put a
large, richly pixel-detailed curved Earth hemisphere in the lower quarter with
connected geography, ocean depth, atmosphere, clouds, night lights, and open
ocean. Above it, use a layered starfield, bounded nebula ribbon, distant galaxy,
satellites, and sparse authored debris. Show one medium asteroid beside the
tower with a fixed warning route, sparse curved gravity grid, direction arrow,
and small `GRAVITY PULL` label. The warning must not track a dropped block.
Include only `SPACE II`, `ASTEROID FIELD`, `GOAL 780M`, and `GRAVITY PULL`.
Avoid meteor spam, photoreal Earth, generic dashboard chrome, and tracking
telegraphs.

## 05 — Reactive World asset board

Create a clean 16:9 pixel production board derived exactly from the five
coordinated concepts. Use five columns labeled `CAVES`, `FOREST`, `SKY`,
`AURORA`, and `SPACE`. In each, show a six-color palette, material cluster,
distant silhouette, habitat creature, ambient family, gameplay telegraph,
object/power-up, and biome block edge. Add a footer labeled `SHARED UI` with
pause, coin, progress, mint action, warning plate, and mascot. Preserve consistent
16–18 px vocabulary, integer scaling, dark-navy cells, and breathing room. Avoid
an asset-store collage, unrelated sprites, long annotations, blur, and overlap.
